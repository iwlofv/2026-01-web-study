# 첫 안내 스크립트

아래 문서는 첫 시간에 학생들에게 바로 읽거나, 공지로 올릴 수 있는 짧은 진행 스크립트입니다.

## 오프닝
안녕하세요. 이번 학기 웹 프로젝트 스터디는 HTML, CSS, JavaScript를 직접 수정하면서 배우는 방식으로 진행합니다. 단순히 이론만 보는 게 아니라, 실제 파일을 열고 바꾸고 실행해보는 흐름에 익숙해지는 것이 목표입니다.

## 운영 방식 설명
이번 스터디는 하나의 공용 레포를 기준으로 진행하지만, 학생들은 그 레포를 각자 계정으로 `fork`해서 작업합니다. 즉 공용 레포 `main`에서 직접 작업하지 않고, 본인 fork 레포의 `main`에서 작업한 뒤 Pull Request로 제출합니다. 이렇게 하면 운영자는 제출 여부와 변경 내용을 보기 쉽고, 학생들은 충돌을 줄이면서 피드백을 받기 편합니다. 중요한 점은 학생 제출 PR은 확인용이며, 공용 레포 `main`에는 merge하지 않는다는 점입니다.

여러분 작업물은 본인 fork 레포의 `projects/<github-id>/` 폴더 아래에 넣으면 됩니다. 다른 사람 폴더는 수정하지 않고, 본인 폴더만 관리하면 됩니다. 예를 들면 `pahaha404`가 fork한 레포에서 `projects/pahaha404/week1/`, `projects/pahaha404/week2/`를 계속 채워가면 됩니다. 반대로 공용 레포 `main` 브랜치에는 학생 폴더를 남기지 않습니다.

## 실습 설명
첫 주차에는 `week1/` 폴더의 예제 파일을 실행해볼 겁니다. 여기서 이름, 만들고 싶은 사이트, 사용할 AI 도구, GitHub 아이디를 채우고 버튼 문구나 색상, 클릭 시 뜨는 문구를 바꿔보면 됩니다.

## AI 활용 안내
모르는 게 생기면 AI에게 바로 질문해도 됩니다. 다만 질문할 때는 막연하게 물어보기보다, 어떤 파일에서 무엇을 바꾸고 싶은지 구체적으로 말하는 게 중요합니다.

예를 들면:
- `index.html에서 제 이름 부분을 바꾸고 싶어요. 어디를 수정하면 되나요?`
- `style.css에서 버튼 색을 초록색으로 바꾸고 싶어요.`
- `script.js에서 버튼 클릭 시 문구를 바꾸고 싶어요.`

## 마무리
오늘 목표는 완벽한 결과물을 만드는 게 아니라, 파일을 열고 수정하고 실행해보는 흐름을 익히는 것입니다. 막히는 부분이 있으면 바로 질문하고, 본인 fork 레포에서 작업한 뒤 Pull Request로 제출하면 됩니다. 제출 PR은 운영자가 확인한 뒤 merge하지 않고 닫습니다.
