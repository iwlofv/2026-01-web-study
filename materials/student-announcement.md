# 웹 프로젝트 스터디 안내

안녕하세요. 2026학년도 1학기 KOSS 웹 프로젝트 스터디를 시작합니다.

이 스터디에서는 HTML, CSS, JavaScript 기초를 직접 수정하면서 웹 페이지를 만드는 흐름을 익힙니다. AI 도구를 함께 활용해서, 모르는 부분을 질문하고 스스로 수정해보는 방식으로 진행합니다.

## 진행 방식
- 모든 실습과 과제는 이 레포 안에서 관리합니다.
- 주차별 자료는 `week1/`, `week2/` 등에 정리됩니다.
- 각자 작업물은 본인 브랜치의 `projects/<github-id>/` 폴더 아래에 정리합니다.
- 작업은 본인 GitHub 아이디 이름의 브랜치에서 하고, 제출은 Pull Request로 진행합니다.
- `main` 브랜치에는 학생 제출물을 올리지 않습니다.

## 첫 주차에 할 일
- `week1/` 폴더의 starter files를 실행합니다.
- 자기소개 문구, 버튼 문구, 색상, 클릭 시 뜨는 문구를 수정합니다.
- 수정한 결과물을 본인 제출 폴더에 정리한 뒤 Pull Request를 올립니다.

## 꼭 지켜주세요
- 폴더 구조는 임의로 바꾸지 않습니다.
- `main` 브랜치에 직접 push하지 않습니다.
- 학생 제출 PR은 기본적으로 merge하지 않습니다.
- 다른 사람 폴더는 수정하지 않습니다.
- 커밋 메시지는 작업 내용을 알 수 있게 작성합니다.
- 막히면 혼자 오래 붙잡지 말고 질문합니다.

## 제출 위치 예시

```text
projects/
  pahaha404/
    week1/
```

브랜치 예시:

```text
pahaha404
```

PR 예시:

```text
pahaha404 -> main
리뷰 후 close
```

## 참고
- 실행은 `week1/index.html`을 브라우저로 열거나 VS Code `Live Server`를 사용하면 됩니다.
- 과제 제출 방법은 `materials/submission-guide.md`를 확인하세요.
